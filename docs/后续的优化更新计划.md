# GoIM 后续优化更新计划

## 1. 目的

本文用于登记前后端联调完成后的体验优化、兼容性改进、非阻断缺陷与后续功能迭代。每项工作在实施前明确范围、优先级和验收标准；完成后补充实施记录与回归结果，避免优化项在日常使用反馈中遗漏。

联调期间已发现并关闭的问题仍以《前后端联调问题记录》为准，本文件只记录联调结束后的新事项。

## 2. 优先级与状态

| 优先级 | 含义 | 建议处理时机 |
| --- | --- | --- |
| P0 | 安全、数据或服务不可用问题 | 立即处理，暂停发布 |
| P1 | 核心使用体验或发布范围内的关键兼容性问题 | 发布前完成 |
| P2 | 明显影响体验但存在替代方案的问题 | 排入近期迭代 |
| P3 | 细节体验、视觉或低频场景优化 | 统一规划处理 |

| 状态 | 含义 |
| --- | --- |
| 待评估 | 已收集，尚未确认范围或方案 |
| 待开发 | 方案和验收标准已明确 |
| 开发中 | 正在修改代码或配置 |
| 待验证 | 已实现，等待自动化或人工回归 |
| 已完成 | 实现及验收均已完成 |
| 暂缓 | 当前不处理，保留原因和重新评估条件 |

## 3. 记录规则

- 日常使用中发现的问题或优化建议先登记，再决定是否立即处理。
- P0、P1 项处理后必须完成受影响业务的回归验证；P2、P3 项至少验证原始复现路径。
- 涉及界面布局的事项应同时验证桌面宽屏、桌面小窗口与手机浏览器。
- 每次实施记录写明日期、改动范围、验证结果和关联提交；不记录密码、令牌等敏感信息。

## 6. 响应式适配建议（OPT-001）

### 范围

- **桌面宽屏**：保留当前多栏工作台布局。
- **桌面小窗口 / 平板横屏**：压缩侧栏和间距，必要时收起次要信息区域。
- **手机浏览器**：采用单栏层级导航：会话列表进入聊天页，资料卡、群资料和设置以独立页面、抽屉或底部弹层展示。

### 需要重点验证的场景

1. 登录、注册、退出和单设备登录状态。
2. 私聊、群聊、收发消息、未读数、消息输入及移动端软键盘。
3. 联系人、好友申请、群创建与群管理。
4. 朋友圈浏览、发布、点赞和评论。
5. 个人资料、头像上传、通知设置和浏览器权限提示。
6. 横竖屏切换、页面刷新、网络断开重连及 WebSocket 状态恢复。

### 实施原则

- 不改变现有后端接口和业务规则，优先在前端布局、导航和交互层完成适配。
- 使用明确的响应式断点与统一的布局组件，避免为单个页面堆叠临时样式。
- 先完成核心聊天闭环，再扩展至联系人、朋友圈和设置页面。
- 每完成一个页面或流程，即在桌面宽屏、窄桌面和至少一种手机浏览器上回归验证。
