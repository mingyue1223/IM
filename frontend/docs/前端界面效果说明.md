# GoIM 前端界面效果说明（首期）

## 1. 产品定位与范围

首期交付一个仅面向桌面浏览器的即时通讯 Web 应用。信息架构借鉴微信桌面端的三栏布局，视觉风格采用全浅色、克制、宽松留白的 macOS 应用感。

首期包含：认证、私聊文字消息、联系人与好友申请、群聊与完整群管理、文字朋友圈、个人设置。首期不做移动端适配、深色模式、图片/视频/表情消息、聊天图片上传或朋友圈图片动态。

媒体能力需等待后端补齐通用媒体上传、安全校验与资源管理后，再作为独立的前后端协同迭代实现。

## 2. 视觉与动效原则

### 视觉

- 以暖白、雾灰和极浅的分层背景为主；品牌色只用于主操作、激活态、未读数与状态提示。
- 使用系统字体栈，清晰优先；不使用夸张的装饰字体。
- 用留白、细微阴影、半透明材质和圆角区分层级，避免厚重边框与高饱和大面积色块。
- 默认保持较低信息密度：内容区宽阔、消息与列表项间距舒适，阅读和输入均有呼吸感。
- 仅维护浅色主题。

### 动效

- 动效用于说明层级变化、空间来源和操作反馈；短时、柔和、有惯性，不以炫技为目的。
- 导航切换使用淡入与轻微位移；选中态使用平滑背景滑块。
- 列表新增或排序变化使用布局过渡，避免内容突然跳动。
- 抽屉从右侧滑入并配合低强度背景遮罩；弹窗和反馈提示使用渐显/渐隐。
- 消息、新消息角标、点赞计数只采用轻量反馈。
- 尊重系统的“减少动态效果”偏好：启用时关闭或显著缩短非必要动画。

## 3. 应用框架

主界面为三栏结构：

```text
导航窄栏 | 当前模块列表栏 | 主内容区（聊天、详情或动态）
```

- 导航窄栏：聊天、联系人、朋友圈、设置。激活项显示极浅品牌色的圆角底。
- 列表栏：随模块切换显示会话、联系人或其他模块的导航内容。
- 主内容区：显示当前聊天、联系人名片、朋友圈时间流或设置内容。
- 联系人/群资料使用按需展开的右侧抽屉，不默认挤压聊天内容。
- 首期仅支持桌面尺寸；窗口不足以容纳三栏时显示最小宽度提示，不提供移动布局。

## 4. 页面与交互

### 4.1 登录与注册

- 居中的轻质卡片、克制的品牌标识和简短欢迎语。
- 登录与注册在同一卡片内平滑切换，不做整页跳转。
- 登录成功后应用壳淡入并默认进入聊天页。
- Token 失效时清理登录态并回到登录页，同时给出明确提示。

### 4.2 聊天

- 会话栏包含搜索、头像、名称、最后消息、时间、未读数和选中态。
- 无会话时在主区显示空状态，并引导用户到联系人页发起会话。
- 聊天顶部显示头像、名称、必要状态和资料抽屉入口。
- 消息按日期分隔；己方消息使用低饱和品牌浅色气泡，对方使用白色气泡。消息流有较宽松的垂直间距。
- 输入区为独立圆角面板，只支持多行文字输入；`Enter` 发送，`Shift + Enter` 换行。发送按钮仅在有内容时突出。
- 图片、表情、文件按钮不进入首期界面；不应呈现无法完成的上传入口。
- 支持本地待发送、服务端确认、失败重试、消息撤回占位、新消息提示、离线同步、自动滚动和断线重连状态。
- 用户正在查看历史消息时不强制滚到底部；收到新消息时显示“有新消息”提示，由用户主动回到底部。

### 4.3 联系人和好友申请

- 左侧为搜索、好友申请入口和按字母分组的联系人列表。
- 右侧为轻量社交名片：大头像、昵称、后端可提供的基础资料及“发消息”主按钮。
- 点击“发消息”后切换至聊天模块，并选中相应会话。
- 好友申请在独立列表或抽屉中处理，展示申请人和验证信息；接受/拒绝后以柔和状态反馈替代原按钮。
- 删除好友和拉黑收纳到更多菜单，均需二次确认。

### 4.4 群聊与群管理

- 群聊消息区沿用私聊的视觉与发送体验。
- 群资料位于右侧抽屉，包含群头像、群名称、成员数、公告和成员网格；可进入完整成员列表。
- 首期提供完整管理：创建群、编辑群信息、邀请成员、移除成员、退出群、查看成员、调整成员角色。
- 角色为群主、管理员、普通成员；界面严格按当前用户权限呈现操作，不向普通成员暴露管理入口。
- 移除成员、退出群、角色调整等高风险操作必须确认；群主无法直接退群时需显示明确引导。

### 4.5 朋友圈

- 定位为熟人间、低打扰的文字动态流，不是公开内容广场。
- 时间流阅读宽度收敛、留白充足。每条动态显示头像、昵称、发布时间、正文与低干扰操作区。
- 发布入口位于页面顶部，提供正文和公开/仅好友/私密可见性选择。
- 点赞采用轻量颜色与计数过渡；评论行内展开，避免复杂浮层。
- 只支持文字动态；图片动态及其上传链路不进入首期。
- 包含首次加载、空动态、加载更多与请求失败状态。

### 4.6 设置

- 采用类似 macOS“设置”应用的分组列表与右侧内容面板。
- 包含个人资料展示、头像上传、通知开关、消息预览开关、免打扰会话列表、退出登录和关于信息。
- 首期个人资料以展示和头像上传为主；后端未提供的昵称、签名编辑不在界面中承诺。
- 保存后使用不打断操作的轻提示反馈。

## 5. 通用状态与可用性

- 每个数据区域必须定义加载骨架、空状态、错误状态与重试入口。
- WebSocket 断线应显示非阻断的连接状态；重连成功后同步消息。收到单设备登录踢出事件后停止重连并转至登录页。
- 破坏性操作：删除好友、拉黑、退出群、移除成员、变更角色均需二次确认。
- 控件具有可见焦点、键盘可操作性与足够的对比度；图标按钮提供可理解的文字提示。

## 6. 首期验收边界

首期的关键体验是：用户可以登录，添加好友并发起私聊，可靠地发送/同步/撤回文字消息，创建并完整管理群聊，发布和互动文字朋友圈，以及管理通知和免打扰设置。所有非文字媒体能力均不纳入首期验收。
