# GoIM 前后端联调实施计划

## 1. 文档目的

本文用于指导 GoIM Web 客户端与 Go 后端在 `dev` 分支上的真实环境联调。执行时按阶段逐项检查、记录结果；当前一阶段达到退出条件后，才进入下一阶段。

联调重点是验证“浏览器界面 → HTTP / WebSocket → 后端服务 → MySQL、Redis、RabbitMQ → 浏览器状态更新”的完整业务闭环。后端接口的既有单元测试和 E2E 测试仍需回归，但不能替代浏览器中的真实操作验证。

## 2. 联调范围与完成标准

### 2.1 本轮范围

- 注册、登录、Token 刷新、登录态恢复与退出。
- 好友申请、接受、拒绝、好友列表、删除、拉黑与解除拉黑。
- 群组创建、资料更新、成员管理、角色权限、退出群组。
- 私聊和群聊文字消息、发送确认、实时接收、已读、撤回、失败重试。
- 断线重连、离线同步、会话摘要、未读数、消息去重和单设备登录。
- 文字朋友圈发布、可见性、Feed、点赞、评论和分页。
- 通知设置、消息预览、会话免打扰、头像上传与访问。
- 加载、空数据、业务错误、认证失效、网络中断等前端反馈。

### 2.2 本轮不包含

- 聊天图片、视频、文件等媒体消息。
- 朋友圈媒体上传。
- 多设备同时在线；当前协议为单设备登录策略。
- 尚未纳入 v1 协议的好友实时通知、在线状态和应用层 JSON 心跳。
- 生产部署、生产域名、HTTPS/WSS 和大规模性能压测。

### 2.3 总体完成标准

满足以下条件后，才可认为本轮联调完成：

1. 所有 P0、P1 问题已关闭，P2 问题已关闭或有明确且被接受的处理结论。
2. 下文所有必测项通过；阻塞或不适用项均有原因记录。
3. 三个真实账户可以通过前端完成好友、私聊、群聊和朋友圈主要闭环。
4. 页面刷新、WebSocket 断开重连、Token 失效等场景不会导致重复消息、无限重连或空白页。
5. 后端全量测试、后端 E2E、前端测试和前端生产构建全部通过。
6. 联调修复均已提交到 `dev`，工作区干净，并完成最终回归。

## 10. 阶段 5：私聊实时消息闭环

### 目标

验证 IM 最核心的双端实时收发与消息状态机。

### 主流程

- [x] A、B 均显示 WebSocket“已连接”。
- [x] A 向 B 发送文字，A 消息经历 `pending → sent`，B 只收到一条相同内容。
- [x] B 回复 A，双向实时收发正常，会话预览和时间更新。
- [x] 打开会话后发送 `readAck`，本地未读数清零。
- [x] A 发送后立即撤回，A、B 均显示撤回状态。
- [x] 使用 HTTP 搜索可查询到当前用户参与的对应私聊消息。
- [x] 删除消息后刷新和重新同步时符合既定删除语义。

### 失败与幂等

- [x] 非好友私聊返回明确失败状态，输入内容不导致页面崩溃。
- [x] 使用同一客户端 `msgId` 重试不会产生两条服务端消息。
- [x] 未收到 `serverAck` 的消息可重试，并复用原客户端消息 ID。
- [x] 已收到 `serverAck` 的消息不会因重连再次发送。
- [x] 非发送者撤回和超过两分钟撤回被拒绝，原消息保持不变。
- [x] 连续快速发送多条消息时 ID 唯一、顺序合理、无遗漏和重复。

### 退出条件

- 双向私聊、确认、已读、撤回、失败和重试全部通过。
- 浏览器 UI、Redis/MySQL 最终数据及后端日志之间无明显矛盾。

## 11. 阶段 6：群聊、断线重连与离线同步

### 目标

### 群聊

- [x] A 发送群消息，A 收到含 `groupSeq` 的确认，B/C 各只收到一条。
- [x] B、C 分别回复，三端顺序和内容一致。
- [x] 非成员发送群消息返回明确失败；已退群成员不再收到新群消息。
- [x] 群消息撤回后所有在线成员同步更新。

### 断线与同步

- [x] B 断网或关闭页面，A 连续发送多条私聊和群聊消息。
- [x] B 恢复连接后自动发起 `syncReq`，离线消息完整出现。
- [x] `syncBatch.hasMore=true` 时继续分页，下一页携带 `syncTime + syncMsgId` 复合游标。
- [x] 同一毫秒内多条消息在小 `batchSize` 下仍无重复、无遗漏。
- [x] `convSync` 后会话预览、排序和未读数与服务端一致。
- [x] 实时消息与同步消息交界处按 `msgId/serverMsgId` 去重。
- [x] 网络反复切换时采用退避重连，不出现多条并行 WebSocket 或无限高频重试。

### 退出条件

- 三端群聊一致，断线期间的私聊和群聊均可恢复。
- 分页同步、去重、会话摘要和未读数全部正确。
