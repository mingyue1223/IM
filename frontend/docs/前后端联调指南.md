# GoIM 前后端联调指南

## 一次性准备

在 `backend/` 下执行：

```powershell
docker compose up -d
Copy-Item configs/config.local.example.yaml configs/config.yaml
go run ./cmd/server -c configs/config.yaml
```

基础服务地址：MySQL `localhost:13306`、Redis `localhost:16379`、RabbitMQ `localhost:5673`，管理台为 `http://localhost:15673`（`guest` / `guest`）。默认本地后端健康检查为 `http://localhost:18080/health`，Swagger 为 `http://localhost:18080/swagger/index.html`。

> `configs/config.yaml` 已被 Git 忽略。已有本地配置时请只比较并合并 `config.local.example.yaml`，不要覆盖现有文件。

## 前端环境变量

复制 `frontend/.env.example` 为前端项目的 `.env.local`：

```env
VITE_API_BASE_URL=http://localhost:18080/api/v1
VITE_WS_URL=ws://localhost:18080/ws
VITE_STATIC_BASE_URL=http://localhost:18080
```

- HTTP 请求基址使用 `VITE_API_BASE_URL`。
- WebSocket 使用 `${VITE_WS_URL}?token=<accessToken>`。
- 上传成功的 `url`（例如 `/uploads/avatars/...`）需与 `VITE_STATIC_BASE_URL` 拼接。

## 联调账户

可分别注册两个本地账号：

```powershell
Invoke-RestMethod -Method Post http://localhost:18080/api/v1/auth/register -ContentType 'application/json' -Body '{"username":"alice","password":"GoimDev123!"}'
Invoke-RestMethod -Method Post http://localhost:18080/api/v1/auth/register -ContentType 'application/json' -Body '{"username":"bob","password":"GoimDev123!"}'
```

随后通过 `/auth/login` 取得令牌，并按《前端接口契约》和《WebSocket 协议》完成好友、私聊和群聊联调。响应体统一为 `{ code, message, data }`，业务数据在 `data` 内。

## 每日启动检查

```powershell
docker compose ps
Invoke-RestMethod http://localhost:18080/health
```

若迁移文件新增且 Docker 数据卷已存在，需手动执行对应 SQL；Docker 初始化脚本只在首次创建 MySQL 数据卷时运行。

## 端口冲突

本地示例使用 HTTP `18080` 和 pprof `16060`。如果任一端口被占用，修改本地 `configs/config.yaml` 的 `server.port` 或 `server.pprof_port`，并同步调整前端 `.env.local` 中的 API、WebSocket 和静态资源地址。将 `pprof_port` 设为 `0` 可关闭 pprof。

## 验证与排错

- 接口字段：`frontend/docs/前端接口契约.md`
- 实时消息：`frontend/docs/WebSocket协议.md`
- HTTP 类型：`frontend/goim-api-types.ts`
- WebSocket 类型：`frontend/goim-ws-types.ts`
- 集成回归：`go test ./tests -tags=e2e -count=1 -v -timeout 120s`
